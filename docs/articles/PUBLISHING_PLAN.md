**封面图建议**：终端截图 + 架构图 + 大字标题

图 1：架构全景图
> 你是不是也觉得 AI Agent 很神秘？我花 3 个月从零写了一个，今天把核心设计分享给你🧵

图 2：三种引擎对比
> 很多人以为 Agent 切换行为=改 Prompt。错！ReAct、Plan-Execute、Reflection 是完全不同的控制流，需要不同的引擎类。我的项目里 8 个引擎各自独立，切模式=换类不是换 Prompt。

图 3：断路器
> LLM 会反复用同样参数调同一个失败的工具，浪费 Token。我加了断路器——3 次连续失败自动熔断，30s 冷却。状态跨轮次持久化，Agent 全程记住"这个工具暂时不能用"。

图 4：6 步压缩流水线
> 上下文太长怎么办？不是简单"取最后 N 条"。我做了 6 步流水线：摘要→精简→去重→评分→裁剪→重组，按语义密度保留最重要内容。
