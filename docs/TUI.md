# Xenon TUI 设计与操作说明

> 适用版本：Xenon v0.7.0+

## 定位

TUI 是 Xenon 的展示与交互层，不改变底层的模型路由、8 种推理引擎、工具执行管线或上下文管理方式。v0.7.0 减少边框和辅助信息对正文的干扰，并修复折叠详情重绘导致的界面错位，让输入、回复和运行状态具有稳定的视觉位置。

## 屏幕布局

```text
对话历史和模型回复

  💭 2 步 · 1 个工具  [Ctrl+O 展开详情]

● ReAct 结果
  这里是正常亮度的最终回复。

───────────────────────────────────────────────
  ❯ 用户输入
───────────────────────────────────────────────

  ● deepseek  ·  … · deepseek/deepseek-v4-pro  ·  react  ·  context 5.0%  ·  cache 89%  ·  ¥0.03  ·  tools 1  ·  Ctrl+O details  ·  Shift+Tab mode
```

布局约束：

1. 上边界、输入内容和下边界组成输入区，两条线会随终端宽度延伸。
2. 下边界只负责结束输入区，不承载 API 或模型状态。
3. 状态栏位于下边界下方，由 `prompt_toolkit` 的 `bottom_toolbar` 固定在整个终端屏幕底端。
4. 输入内容超过一行时，输入区向上增长；状态栏位置不变。
5. 终端较窄时，底栏会优先保留 API 状态和上下文用量，再按优先级省略费用、工具计数和快捷键提示，避免折行。

## 视觉层级

| 内容 | 显示方式 |
|------|----------|
| 最终回复 | 无大边框，正常亮度，以 `● Assistant` 或引擎结果名作为轻量锚点 |
| 优化后的 Prompt | 无边框、缩进排版、整体 dim，不与回复争夺注意力 |
| 意图、模式切换和 HTTP 日志 | 辅助信息，dim 显示 |
| 工具调用与探索过程 | 默认只显示步数/工具数摘要，完整内容折叠 |
| API 与会话状态 | 始终位于屏幕底栏，不混入输入框边界 |

`/cost`、`/models` 等结构化命令可以继续使用表格或面板；“无边框”的约束主要针对高频的模型回复、提示词和执行轨迹。

## 快捷键

| 操作 | 快捷键 |
|------|--------|
| 发送输入 | `Enter` |
| 输入换行 | `Shift+Enter` 或 `Alt+Enter` |
| 展开/折叠上一次执行详情 | `Ctrl+O` |
| 循环切换推理范式 | `Shift+Tab` |
| 中断当前引擎 | `Ctrl+C` |
| 退出 Xenon | 空闲时连续两次 `Ctrl+C` |

## 状态栏字段

底栏会按终端宽度选择显示以下字段：

- API 配置/连接状态与 provider。
- 当前或 AutoRouter 激活的模型。
- 推理范式。
- 上下文使用率与 `/compact` 警告。
- DeepSeek 缓存命中率和预估人民币费用。
- 工具调用数、会话时长和快捷键提示。

## 回退模式

默认交互界面使用 `prompt_toolkit`。当设置 `XENON_NO_PT=1` 或当前终端不支持时，Xenon 会回退到普通输入模式；对话和工具功能仍可用，但底栏不保证能固定在屏幕最底端。

## 实现位置

- `xenon/repl/repl.py`：输入布局、无边框回复、辅助文本和 `Ctrl+O` 切换。
- `xenon/repl/status_bar.py`：完整宽度边界、固定底栏和窄屏字段取舍。
- `xenon/engine/callbacks.py`：执行轨迹采集、折叠摘要和敏感参数脱敏。
- `tests/test_tui_theme.py`：输入线宽度、底栏定位、无边框渲染回归测试。
